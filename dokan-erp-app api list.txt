#####################DokanERP API LIST######################

###################Customer Registration / Customer Master APIs###################

Customer vs Owner:
- customer is the buyer/client of a shop
- owner is the merchant/shop owner who creates and manages the shop
- customer creation does not create login access
- owner registration creates the merchant account and shop

Rule:
- customer master data is global
- customer create is not shop-wise
- future customer purchase / due / payment data should be shop-wise

1. POST /app/api/customers
Request:
```json
{
  "companyOrPersonName": "Karim Store Buyer",
  "mobile": "01712000001",
  "address": "Mirpur, Dhaka",
  "shortNote": "Regular retail customer"
}
```

Optional fields:
- `customerCode`
- `name`
- `email`
- `notes`
- `status`

Response:
```json
{
  "message": "Customer created successfully.",
  "customer": {
    "id": "customer_xxx",
    "customerCode": "KARIMS-123456",
    "name": "Karim Store Buyer",
    "companyOrPersonName": "Karim Store Buyer",
    "mobile": "01712000001",
    "email": null,
    "address": "Mirpur, Dhaka",
    "shortNote": "Regular retail customer",
    "notes": "Regular retail customer",
    "status": "ACTIVE",
    "statusLabel": "ACTIVE",
    "createdAt": "2026-06-11T06:00:00.000Z",
    "updatedAt": "2026-06-11T06:00:00.000Z"
  }
}
```

Duplicate response:
```json
{
  "message": "Customer already exists. Duplicate customer add is blocked.",
  "customer": {
    "id": "customer_xxx",
    "customerCode": "CUS-001",
    "name": "Karim Store Buyer",
    "companyOrPersonName": "Karim Store Buyer",
    "mobile": "01712000001"
  }
}
```

2. GET /app/api/customers
Example:
```http
GET http://localhost:4000/app/api/customers
```

3. GET /app/api/customers/:customerIdOrCode
Example:
```http
GET http://localhost:4000/app/api/customers/CUS-001
```

Customer notes:
- this section is only for customer master data
- customer is not the shop owner
- customer does not use `register-owner-draft`
- customer is created globally, then customer purchase/due/payment is tracked shop-wise

###################Shop-wise Customer Sales / Due / Payment APIs###################

These APIs are shop-wise and use the global customer master as the customer source.

1. GET /app/api/customers?shopId=:shopIdOrCode
Example:
```http
GET http://localhost:4000/app/api/customers?shopId=SHOP-001
```

Response shape:
- `shop`
- `stats.totalSales`
- `stats.totalPaid`
- `stats.totalDue`
- `customers[].totalSales`
- `customers[].totalPaid`
- `customers[].due`

2. POST /app/api/customers/sales
Request:
```json
{
  "shopId": "shop_xxx",
  "customerId": "customer_xxx",
  "invoiceNo": "SAL-240611-001",
  "paidAmount": 500,
  "paymentMethod": "cash",
  "notes": "Partial payment received",
  "items": [
    {
      "masterProductId": "product_xxx",
      "quantity": 2,
      "salePrice": 150
    },
    {
      "masterProductId": "product_yyy",
      "quantity": 1,
      "salePrice": 200
    }
  ]
}
```

Notes:
- creates a shop-wise customer sale
- creates customer ledger entries
- if `paidAmount > 0`, also creates a customer payment entry

3. GET /app/api/customers/:customerIdOrCode?shopId=:shopIdOrCode
Example:
```http
GET http://localhost:4000/app/api/customers/CUS-001?shopId=SHOP-001
```

Response shape:
- `shop`
- `customer.finance`
- `customer.recentSales`
- `customer.recentPayments`
- `customer.recentTransactions`

4. GET /app/api/customers/:customerIdOrCode/sales?shopId=:shopIdOrCode
Example:
```http
GET http://localhost:4000/app/api/customers/CUS-001/sales?shopId=SHOP-001
```

5. POST /app/api/customers/:customerIdOrCode/payments
Request:
```json
{
  "shopId": "shop_xxx",
  "amount": 300,
  "paymentMethod": "cash",
  "referenceNo": "COLLECT-240611-001",
  "notes": "Due collection"
}
```

Notes:
- payment cannot be greater than the current outstanding due

6. GET /app/api/customers/:customerIdOrCode/ledger?shopId=:shopIdOrCode
Example:
```http
GET http://localhost:4000/app/api/customers/CUS-001/ledger?shopId=SHOP-001
```

Ledger note:
- debit increases customer due
- credit reduces customer due

#####################Owner Registration Flow######################

Registration rules:
- Password must be at least 4 characters
- PIN must be exactly 4 digits
- Mobile number must be unique
- Password and confirm password must match
- PIN and confirm PIN must match
Recommended flow:
1. Registration page-1
   - owner name
   - mobile number
   - password
   - confirm password
2. Registration page-2
   - shop name
   - shop address
   - shop category
   - shop location from live detect
   - if live location fails, allow manual location entry
3. Submit registration data
   - save owner and shop data as pending
   - do not activate account yet
4. Send OTP
   - send OTP to the provided mobile number
   - OTP expires after 2 minutes
5. Verify OTP
   - if OTP is correct, mark mobile as verified
   - if OTP is wrong or expired, show resend option
   - resend or renew OTP after 2 minutes
6. Setup PIN
   - enter 4-digit PIN
   - confirm 4-digit PIN
7. Complete registration
   - activate owner account
   - activate shop
   - save PIN
8. Welcome page
   - show success message
   - continue to dashboard or first-time setup
Suggested API list:
1. POST /app/api/auth/register-owner-draft
Request:
{
  "name": "Rahim Uddin",
  "mobile": "01712345670",
  "password": "1234",
  "confirmPassword": "1234",
  "shopName": "Rahima Grocery Store",
  "shopAddress": "House 12, Road 5, Mirpur, Dhaka, Bangladesh",
  "shopCategory": "Grocery",
  "shopLocation": "Mirpur, Dhaka",
  "latitude": 23.8103,
  "longitude": 90.4125
}
Response:
{
  "message":"Registration draft created successfully.",
  "registrationId":"cmq7lb0c70000lx8zrgjjy4zk",
  "draft":{
            "id":"cmq7lb0c70000lx8zrgjjy4zk",
            "mobile":"01712345670",
            "shopName":"Rahima Grocery Store",
            "status":"PENDING",
            "expiresAt":"2026-06-10T05:19:59.670Z"
          }
}
2. POST /app/api/auth/send-otp
Request:
{
  "registrationId": "cmq7lb0c70000lx8zrgjjy4zk",
  "mobile": "01712345670"
}
Response:
{"message":"OTP sent successfully.","registrationId":"cmq7lb0c70000lx8zrgjjy4zk","expiresAt":"2026-06-10T04:58:30.147Z","demoOtp":"4219"}
3. POST /app/api/auth/verify-otp

  password >= 4

###################Mobile OTP Login Flow###################

Simple journey:
1. Mobile number screen
2. OTP screen
3. Dashboard

Rules:
- Login uses only mobile number
- OTP expires after 2 minutes
- Only existing active mobile users can log in
- OTP is 4 digits

1. POST /app/api/auth/send-login-otp
Request:
{
  "mobile": "01712345679"
}

Response:
{
  "message": "OTP sent successfully.",
  "loginRequestId": "otp_request_xxx",
  "expiresAt": "2026-06-09T06:20:00.000Z",
  "expiresInSeconds": 120,
  "demoOtp": "1234"
}

2. POST /app/api/auth/verify-login-otp
Request:
{
  "loginRequestId": "otp_request_xxx",
  "mobile": "01712345679",
  "otp": "1234"
}

Response:
{
  "message": "Login successful.",
  "authenticated": true,
  "redirectTo": "/shop/dashboard",
  "role": "SHOP_OWNER",
  "appType": "MOBILE",
  "user": {
    "id": "user_xxx",
    "name": "Rahim Uddin",
    "mobile": "01712345679"
  },
  "shop": {
    "id": "shop_xxx"
  }
}

Testing steps:
1. Call send-login-otp with a mobile number that already exists in the users table
2. Copy loginRequestId from the response
3. Copy demoOtp from the response or backend console log
4. Call verify-login-otp with loginRequestId, mobile, and otp
5. If OTP is correct, login is completed and dashboard redirect is returned

###################Create Purchase For Specific Shop###################

Purpose:
- create supplier purchase
- add purchase items from master product ids
- auto attach those products to shop_products if not already configured
- support `Cash`, `bKash`, `Nagad`, `Card`, and `Due`

API:
- POST /app/api/purchases

Example request for cash:
```json
{
  "shopId": "RAHIMA677141",
  "supplierId": "supplier_xxx",
  "invoiceNo": "PO-10032",
  "purchaseDate": "2026-06-11T10:30:00.000Z",
  "paymentMethod": "CASH",
  "paidAmount": 2500,
  "notes": "আজকের ক্রয়",
  "items": [
    {
      "masterProductId": "prod_rice",
      "quantity": 24,
      "purchasePrice": 56
    }
  ]
}
```

Example request for due:
```json
{
  "shopId": "RAHIMA677141",
  "supplierId": "supplier_xxx",
  "invoiceNo": "PO-10033",
  "paymentMethod": "DUE",
  "paidAmount": 0,
  "items": [
    {
      "masterProductId": "prod_oil",
      "quantity": 12,
      "purchasePrice": 145
    }
  ]
}
```

Example request for bKash / Nagad:
```json
{
  "shopId": "RAHIMA677141",
  "supplierId": "supplier_xxx",
  "paymentMethod": "BKASH",
  "paidAmount": 3200,
  "paymentDetails": {
    "senderNumber": "01712345678",
    "transactionId": "BKX99887766"
  },
  "items": [
    {
      "masterProductId": "prod_rice",
      "quantity": 10,
      "purchasePrice": 50
    }
  ]
}
```

Example request for card:
```json
{
  "shopId": "RAHIMA677141",
  "supplierId": "supplier_xxx",
  "paymentMethod": "CARD",
  "paidAmount": 3200,
  "paymentDetails": {
    "cardHolderName": "Abdul Karim",
    "cardLast4": "4589",
    "cardType": "Visa",
    "approvalCode": "APR77821"
  },
  "items": [
    {
      "masterProductId": "prod_rice",
      "quantity": 10,
      "purchasePrice": 50
    }
  ]
}
```

Validation rules:
- `CASH`: no extra input required
- `BKASH` / `NAGAD`: `senderNumber` and `transactionId` required
- `CARD`: `cardHolderName`, `cardLast4`, `cardType`, and `approvalCode` or `transactionId` required
- `DUE`: `paidAmount` must be `0`

Success response shape:
```json
{
```

###################Salesman Password + OTP Login Flow###################

- OTP expires after 2 minutes

  - POST /app/api/auth/salesmans-login-otp
